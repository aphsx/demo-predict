# ข้อมูลบริษัทและระบบ 1MOBY CRM

## เกี่ยวกับ 1MOBY
1MOBY เป็นผู้ให้บริการ SMS Marketing Platform สำหรับธุรกิจในประเทศไทย
ลูกค้าซื้อเครดิต SMS แล้วใช้ส่งข้อความหาลูกค้าของตนเอง

## ประเภทลูกค้า
- **paid** — ลูกค้าที่ซื้อแพ็กเกจจริง มีการจ่ายเงิน
- **trial** — ลูกค้าทดลองใช้งาน ยังไม่ได้ซื้อ

## คำศัพท์สำคัญ

### Churn
หมายถึงลูกค้าที่หยุดใช้บริการ ไม่ต่ออายุ หรือไม่ซื้อเครดิตเพิ่ม
- **churn_probability** — ความน่าจะเป็นที่จะ churn (0.0 = ปลอดภัย, 1.0 = เสี่ยงสูงมาก)
- **risk_tier** — High (≥60%), Medium (30–60%), Low (<30%)

### LTV (Life Time Value)
ยอดเงินรวมที่ลูกค้าจ่ายตลอดช่วงที่ใช้งาน คำนวณจาก `total_amount_paid`
ลูกค้า LTV สูง = มีคุณค่าต่อบริษัทมาก ควรดูแลเป็นพิเศษ

### RFM Segment
วิธีแบ่งกลุ่มลูกค้าตาม Recency / Frequency / Monetary:
- **Champions** — ซื้อบ่อย ยอดสูง เข้าใช้งานล่าสุด → ลูกค้าดีที่สุด
- **Loyal** — ซื้อสม่ำเสมอ ยอดปานกลาง
- **Potential** — มีศักยภาพ แต่ยังไม่ถึงระดับ Loyal
- **At Risk** — เคยดีแต่กำลังหายไป → ต้องดูแลด่วน
- **Lost** — หยุดใช้งานนานแล้ว
- **Low Spender** — ซื้อน้อย ใช้งานน้อย

### SHAP Values
ค่าที่อธิบายว่า feature ไหนส่งผลต่อการ predict churn มากที่สุดสำหรับลูกค้าแต่ละคน

## แพ็กเกจ SMS
ลูกค้าซื้อเครดิต SMS เป็นแพ็กเกจ เช่น:
- แพ็กเกจ SMS ทั่วไป (credit_type: standard)
- แพ็กเกจ SMS Premium (credit_type: premium)
- เครดิต Email แยกต่างหาก

## กระบวนการ Predict Churn
1. Admin อัพโหลดไฟล์ข้อมูลลูกค้าและประวัติการซื้อ
2. ระบบรัน Machine Learning Model (XGBoost)
3. ผลออกมาเป็น churn_probability, risk_tier, rfm_segment, risk_factor, recommended_action
4. Admin ใช้ AI Chat วิเคราะห์ผลและวางแผน retention

## Recommended Actions (คำแนะนำที่ระบบ generate)
- **โทรหาทันที** — ลูกค้าเสี่ยงสูง LTV สูง
- **ส่ง SMS ด่วน** — ใกล้หมดอายุ ยังไม่ต่ออายุ
- **เสนอโปรโมชั่น** — ซื้อน้อยลง อาจต้องการแรงจูงใจ
- **ติดตาม** — เสี่ยงปานกลาง ยังพอมีเวลา
- **ปล่อยผ่าน** — LTV ต่ำ ไม่คุ้มลงทุนมาก

## ทีมที่ใช้ระบบนี้
- **Admin / Sales** — ดู top risk, วางแผน retention call
- **Management** — ดูภาพรวม KPI, revenue at risk
- **Data Team** — อัพโหลดข้อมูล, ดู model performance

## คำถามที่พบบ่อย
**Q: ลูกค้า High Risk ทุกคนต้องโทรไหม?**
A: ไม่จำเป็น ควร prioritize ตาม LTV × (1 - churn_probability) — คนที่มีมูลค่าสูงและเสี่ยงสูงควรได้รับการดูแลก่อน

**Q: days_since_last_access กี่วันถือว่าน่ากังวล?**
A: >30 วัน = เริ่มน่าเป็นห่วง, >60 วัน = น่ากังวลมาก, >90 วัน = เสี่ยงสูงมากที่จะ churn

**Q: downgraded หมายความว่าอะไร?**
A: ลูกค้าซื้อเครดิตน้อยกว่าครั้งก่อน แสดงว่าการใช้งานหรืองบประมาณลดลง เป็น signal เตือนภัย
